## 🔍 상세 코드 및 설정 파일


### [01번 문항] 백엔드 API 명세서

### 1. 게시글 목록 조회
전체 게시글의 리스트를 조회합니다.

* **URL:** `/board/list`
* **Method:** `GET`
* **Description:** 등록된 모든 게시글을 최신순으로 반환합니다.

| 파라미터명 | 타입 | 필수 여부 | 설명 |
| :--- | :--- | :---: | :--- |


**Success Response (200 OK):**
```json
[
  {
    "idx": 10,
    "title": "글 제목",
    "content": "글 내용"
  },
  ...
]
```

---

### 2. 게시글 상세 조회
특정 식별값(`idx`)을 가진 게시글의 상세 내용을 조회합니다.

* **URL:** `/board/{idx}`
* **Method:** `GET`
* **Description:** 특정 게시글의 본문 및 상세 정보를 확인합니다.

| 경로 변수 | 타입 | 설명 |
| :--- | :--- | :--- |
| `idx` | Long | 게시글 고유 번호 |

**Success Response (200 OK):**
```json
{
  "idx": 10,
  "title": "글 제목",
  "content": "글 내용"
}
```

---

### 3. 게시글 등록
새로운 게시글을 작성하여 서버에 저장합니다.

* **URL:** `/board/reg`
* **Method:** `POST`
* **Description:** 사용자가 입력한 제목과 본문을 데이터베이스에 저장합니다.

**Request Body:**
```json
{
  "title": "글 제목",
  "content": "글 내용"
}
```

**Success Response (201 Created):**
```json
{
  "status": "success",
  "message": "게시글이 성공적으로 등록되었습니다.",
  "idx": 11
}
```

### [02번 문항] 프론트 엔드 각 페이지 요구사항

### 1. 게시글 목록 페이지 (`/board/list`)
* **기능 요약:** 서버에 저장된 전체 게시글 데이터를 불러와 화면에 출력합니다.
* **UI 구성:**  
    * 게시글 번호(idx), 제목을 포함한 테이블(Table) 구조.
    * 페이지 상단 또는 하단에 게시글 작성을 위한 **[글쓰기]** 버튼 배치.
* **핵심 로직:**
    * **이동:** 제목 클릭 시 상세 페이지(`/board/{idx}`)로, 글쓰기 버튼 클릭 시 등록 페이지(`/board/reg`)로 이동합니다.

### 2. 게시글 상세 페이지 (`/board/{idx}`)
* **기능 요약:** 목록에서 선택한 특정 게시글의 전체 내용을 조회하여 보여줍니다.
* **UI 구성:** 
    * 제목, 본문 내용을 포함하는 상세 뷰 영역.
    * 하단에 목록으로 돌아가기 위한 **[목록보기]** 버튼 배치.
* **핵심 로직:**
    * 잘못된 `idx`로 접근하거나 데이터를 찾을 수 없을 경우 에러 메시지를 노출하거나 목록 페이지로 리다이렉트합니다.

### 3. 게시글 등록 페이지 (`/board/reg`)
* **기능 요약:** 사용자로부터 새로운 게시글 정보를 입력받아 서버에 저장합니다.
* **UI 구성:** 
    * 제목(Input), 내용(Textarea)을 입력할 수 있는 폼(Form) 구조.
    * 데이터 전송을 위한 **[저장]** 버튼.
* **핵심 로직:**
    * **완료 처리:** 등록 성공 시 알림창(alert)을 띄우고 목록 페이지(`/board/list`)로 자동으로 이동합니다.

### [03번 문항] 백엔드 API 호출
* 목록 조회 페이지에서 컴포넌트 마운트 시 `GET /board/list` API를 호출하고 리턴 받은 데이터를 화면에 출력합니다.
* 상세 조회 페이지에서 컴포넌트 마운트 시 URL 경로에 포함된 `idx` 파라미터를 추출하여 `GET /board/{idx}` API를 호출하고 리턴받은 데이터를 화면에 출력합니다.
* 게시글 등록 페이지에서 저장 버튼 클릭 시 `POST /board/reg` API를 통해 데이터를 전송합니다. 해당 데이터는 DB에 저장되어야 합니다.
